# Team Coding Best Practices

1. **Error Handling**: Never use empty catch blocks. Always log errors.
2. **Naming Conventions**: Use snake_case for Python variables, camelCase for JS.
3. **Security**: No hardcoded API keys or secrets.
4. **Comments**: Do not write comments that just describe what the code does; explain *why*.
5. **Performance**: Avoid nested loops where possible (O(n^2)).
6. **Testing**: Every new function must have a corresponding unit test.
